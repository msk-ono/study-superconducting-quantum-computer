# キャリブレーションの概要

超伝導量子ビットは製造ばらつきや環境ノイズの影響を受けやすいため、高精度な制御を行うためには頻繁な**キャリブレーション (校正)** が不可欠です。

## キャリブレーションフロー

一般的に、キャリブレーションは以下の順序で行われます (QDash の `FullCalibration` ワークフローなどを参照)。

1. **基本特性の評価 (Characterization)**
   - 共鳴周波数の探索 (Spectroscopy)
   - ラビ振動 (Rabi Oscillation) による振幅調整
   - $T_1, T_2$ の測定

2. **1量子ビットゲートの調整 (Single Qubit Gate Calibration)**
   - DRAG (Derivative Removal by Adiabatic Gate) パルスの調整
   - ランダム化ベンチマーク (RB) によるゲートフィデリティの評価

3. **読み出しの調整 (Readout Calibration)**
   - 読み出し周波数と振幅の最適化
   - 状態識別 (State Discrimination) のための分類器の学習

4. **2量子ビットゲートの調整 (Two Qubit Gate Calibration)**
   - Cross-Resonance (CR) ゲートのパラメータ調整
   - インターリーブ RB による 2量子ビットゲートフィデリティの評価

## 階層構造

キャリブレーションは依存関係を持つタスクのグラフ (DAG) として表現されます。
例えば、ラビ振動の測定には正確な読み出しが必要であり、2量子ビットゲートの調整には高精度な1量子ビットゲートが必要です。
現在、多くのシステム (QDash等) では、この依存関係をワークフローエンジンで管理し、自動化を進めています。
